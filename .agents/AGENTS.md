# Workspace Rules for Rain Forecast Dashboard

## 🚨 Pair Programming Workflow Rules

1. **Plan First (No Unprompted Edits):** 
   Do not make any modifications to the code or files immediately upon receiving a new instruction from the user.
2. **Review & Explain:** 
   Always repeat the instruction back to the user in Thai, analyze the requirement, and explain step-by-step how it will be implemented (what files, lines, and logic will change).
3. **Wait for Approval:** 
   Wait until the user explicitly says "อนุมัติ", "ทำจริงได้", "ดำเนินการได้เลย" or inputs the shortcut "1" (approve) before performing any file edits or commands that modify the project state. If the user inputs the shortcut "0" (wait/other tasks), do not proceed.
