# ข้อตกลงการเขียนโค้ดร่วมกัน (Pair Programming Workflow)

เพื่อประสานงานและลดข้อผิดพลาดในการแก้ไขระบบ ให้ปฏิบัติตามแนวทางต่อไปนี้ในทุกๆ คำสั่งของคู่ค้าคู่ขนาน (USER):

1. **ไม่แก้ไขโค้ดหรือไฟล์โดยทันที:** เมื่อได้รับคำสั่งใหม่ ให้ทำการวิเคราะห์เชิงทฤษฎีเท่านั้น ห้ามใช้เครื่องมือแก้ไขไฟล์ซอร์สโค้ดใดๆ เด็ดขาดในขั้นตอนแรก
2. **ทบทวนคำสั่งและแจ้งวิธีการทำ:** ตอบกลับด้วยการทวนสิ่งที่ต้องการปรับปรุง (Requirement) เป็นภาษาไทยให้ถูกต้อง พร้อมอธิบายแนวทางขั้นตอนทางเทคนิค (Step-by-step approach) ว่าจะเปลี่ยนไฟล์ไหน ตรงจุดใด และเพื่อผลลัพธ์อะไร
3. **รอการอนุมัติก่อนทำจริง:** รอคอยข้อความสั่งการอนุมัติจากผู้ใช้งาน (เช่น "อนุมัติ", "ทำจริงได้", "ดำเนินการได้เลย") ก่อนที่จะลงมือเรียกใช้เครื่องมือแก้ไขโค้ดหรือรันการเปลี่ยนแปลงใดๆ
